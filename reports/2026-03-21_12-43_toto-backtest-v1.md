# Toto Backtest Engine v1 Report

## Что сделано

- Добавлен `toto/backtest.py` с классом `TotoBacktest`.
- Реализован метод `evaluate(coupons, results)`.
- Реализованы метрики:
  - `max_hits`
  - `avg_hits`
  - `distribution`
  - `ROI`
- Добавлена простая модель выплат:
  - 15 → 100000
  - 14 → 5000
  - 13 → 500
- Добавлены тесты в `tests/test_toto_backtest.py`.

## Backtest Example

- Купоны:
  - `['1' x15]`
  - `['1' x14 + 'X']`
  - `['1' x13 + 'X' + 'X']`
  - `['1' x12 + 'X' + 'X' + 'X']`
- Реальные результаты:
  - `['1' x15]`
- max_hits:
  - `15`

## Weaknesses

- Пока используется фиксированная payout-модель (заглушка), без реальных тиражных коэффициентов.
- Нет анализа чувствительности к размеру пула купонов и стратегии отбора.
- Нет отдельного учета стоимости ставки, кроме базового `stake = len(coupons)`.
- Нет детализации по конкретным матчам, где и почему купон промахнулся.
