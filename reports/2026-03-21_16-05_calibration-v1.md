# REPORT: calibration-v1

## 1. Summary
Реализован единый калибратор вероятностей `ProbabilityCalibrator` на базе `CalibratedClassifierCV` с методом `sigmoid` для multiclass-исходов (1/X/2), с проверками корректности вероятностей и сохранением модели в `data/models/calibrated_model.pkl`.

## 2. Goal
Сделать post-training калибровку вероятностей предобученной модели LightGBM так, чтобы выход `predict_proba` был ближе к реальным вероятностям исходов для сценариев беттинга.

## 3. Scope
- Добавлен новый production-класс калибровки в `core/model/calibrator.py`.
- Добавлены unit-тесты для калибратора в `tests/test_calibrator.py`.
- Добавлен отчет этапа в `reports/`.

## 4. Files Changed
- `core/model/calibrator.py`
- `tests/test_calibrator.py`
- `reports/2026-03-21_16-05_calibration-v1.md`

## 5. Detailed Changes
1. Реализован класс `ProbabilityCalibrator` с API:
   - `fit(model, X_valid, y_valid) -> calibrated_model`
   - `predict_proba(X) -> np.ndarray[n, 3]`
2. В `fit` выполнено:
   - вычисление `log_loss` до калибровки;
   - обучение `CalibratedClassifierCV(..., method='sigmoid', cv='prefit')`;
   - fallback для новых версий sklearn через `FrozenEstimator` (с сохранением sigmoid-подхода);
   - вычисление `log_loss` после калибровки;
   - сохранение `calibrated_model.pkl`.
3. В `predict_proba` добавлены проверки:
   - форма ровно `[n, 3]`;
   - отсутствие `NaN`;
   - сумма вероятностей в каждой строке близка к 1 (`atol=1e-6`).
4. В тестах покрыты требования этапа: shape, NaN-check, корректность суммы, и контроль деградации по `log_loss`.

## 6. Public Contracts / Interfaces
Новый публичный контракт:
- `ProbabilityCalibrator.fit(model, X_valid, y_valid) -> CalibratedClassifierCV`
- `ProbabilityCalibrator.predict_proba(X) -> np.ndarray`
- `ProbabilityCalibrator.save(...) -> None`

Сигнатуры простые и совместимы с текущим пайплайном, где уже используется `predict_proba`.

## 11. Manual Verification
Ручная проверка скриптом на синтетических multiclass-данных:
- raw `log_loss = 0.219612`
- calibrated `log_loss = 0.165195`
- пример распределения вероятностей стал менее overconfident, но остался валидным по сумме.

## 12. Risks / Known Issues
- Для последних версий sklearn `cv='prefit'` в `CalibratedClassifierCV` больше не принимается напрямую, поэтому используется совместимый режим через `FrozenEstimator`.
- На небольших/смещенных валидационных выборках калибровка может не дать улучшения на holdout.

## 13. Deviations from Spec
- Бизнес-логика и архитектурные ограничения соблюдены.
- Реализован технический compatibility fallback для sklearn API, не меняющий выбранный метод калибровки (`sigmoid`) и не добавляющий альтернатив вроде isotonic.

## 14. Next Recommended Step
Интегрировать `ProbabilityCalibrator` в training pipeline после тренировки модели и до сериализации финального артефакта для inference-пайплайна.
