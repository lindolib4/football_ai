# REPORT: model_predictor_v1_strict_inference

## 1. Summary
Реализован строгий inference-класс `ModelPredictor` для получения вероятностей `P1/PX/P2` из `model.pkl` с валидацией фичей и контролем корректности вероятностей.

## 2. Что сделано
- Добавлен класс `ModelPredictor` в `core/model/predictor.py`.
- Реализованы методы:
  - `load(path)` — загрузка `model.pkl`, опциональная загрузка `feature_schema` (`feature_schema.json` или `feature_schema.pkl`).
  - `predict(features)` — строгая проверка входа, вызов `predict_proba`, возврат `{"P1", "PX", "P2"}`.
- Добавлены проверки:
  - все обязательные фичи присутствуют;
  - нет лишних фич;
  - нет `NaN`;
  - вероятности не отрицательны;
  - сумма вероятностей близка к 1.
- Добавлено логирование + исключения для критичных ошибок:
  - отсутствует модель;
  - mismatch фичей;
  - невалидные вероятности.
- Сохранена обратная совместимость через alias `Predictor(ModelPredictor)`.

## 3. Тесты
Добавлен `tests/test_predictor.py` с покрытием:
- успешный путь предсказания `P1/PX/P2`;
- ошибка при отсутствии файла модели;
- ошибка при mismatch фичей;
- ошибка при `NaN` во входе;
- ошибка при невалидной сумме вероятностей.

## 4. Результат проверки
`pytest -q tests/test_predictor.py` — успешно, 5/5 тестов.

## 5. Риски / ограничения
- Если feature schema не найдена рядом с моделью, используется fallback-схема из `FeatureBuilder`.
- Контроль finite (`inf`) для фич не включался отдельным требованием этапа; сейчас блокируется именно `NaN`.

## 6. Deviations
Отклонений от поставленной спецификации этапа нет.
