## 5. Detailed Changes
- `get_draws(name, page=1)` теперь вызывает:
  - `GET /api/v1/community/{name}/drawings`
  - query: `page`
  - возвращает только `id`, `number`, `ended_at`
- `get_draw(draw_id)` теперь вызывает:
  - `GET /api/v1/community/drawing-info/{id}`
  - возвращает:
    - `draw_id`
    - `matches[]` с `name`, `result`, `pool_probs`
- Реализован парсинг pool probabilities:
  - `quotes.pool_win_1 -> P1`
  - `quotes.pool_draw -> PX`
  - `quotes.pool_win_2 -> P2`
  - значения делятся на 100
- Реализовано сохранение raw payload:
  - `data/toto_draws/{draw_id}.json`

## 6. Public Contracts / Interfaces
Новый контракт:
- `TotoAPI.get_draws(name: str, page: int = 1) -> list`
- `TotoAPI.get_draw(draw_id: int) -> dict`

`get_draw` возвращает структуру:
```json
{
  "draw_id": 123,
  "matches": [
    {
      "name": "Team A - Team B",
      "result": "1",
      "pool_probs": {
        "P1": 0.51,
        "PX": 0.27,
        "P2": 0.22
      }
    }
  ]
}
```

## 8. API Behavior
- Используются Totobrief endpoint-ы из ТЗ.
- Поддержан payload c корневым `drawing` и прямым корневым объектом.
- Для списка тиражей поддержаны `drawings`, `data`, а также list в корне как fallback.

## 9. Business Logic Changes
- Нормализация имени матча: сначала `name`, затем fallback `home - away`.
- Нормализация результата: поддержаны `result`, `outcome`, `final_result`, `sign`.
- При отсутствии/некорректности quote полей вероятность = `0.0`.
