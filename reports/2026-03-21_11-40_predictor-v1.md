# REPORT: predictor-v1 + feature-schema

## 1. Summary
Реализована отдельная схема фич (`FeatureSchema`) для строгого контракта инференса, интегрировано сохранение схемы при сохранении модели в trainer, и обновлен `ModelPredictor` для безопасного предикта с валидацией входа/выхода и явным fallback на raw-модель.

## 2. Goal
Добавить безопасный путь инференса: фиксировать список фич обучения, валидировать входные данные и гарантировать корректные вероятности `P1/PX/P2`.

## 3. Scope
- Добавлен новый модуль `core/model/feature_schema.py`.
- Обновлен `core/model/trainer.py` (сохранение feature schema рядом с моделью).
- Обновлен `core/model/predictor.py` (загрузка calibrated/raw + schema validation).
- Обновлены тесты `tests/test_predictor.py` и `tests/test_trainer.py`.

## 4. Files Changed

## 5. Detailed Changes
- `FeatureSchema.save(feature_names, path)` сохраняет JSON-массив строк.
- `FeatureSchema.load(path)` читает и валидирует формат списка строк.
- `FeatureSchema.validate(features, expected)` проверяет missing/extra признаки и выбрасывает исключение с логом.
- В `ModelTrainer.save(...)` после `joblib.dump(...)` сохраняется `feature_schema.json`.
- `ModelPredictor.load(...)`:
  - загружает `calibrated_model.pkl`, если файл существует;
  - иначе использует `model.pkl`;
  - всегда загружает `feature_schema.json`.
- `ModelPredictor.predict(...)`:
  - валидирует вход против схемы;
  - проверяет NaN/inf во входе;
  - вызывает `predict_proba`;
  - валидирует shape `(1, 3)`, отсутствие NaN, диапазон `[0,1]`, сумму вероятностей `≈1`.

## 6. Public Contracts / Interfaces
- `FeatureSchema`:
  - `save(feature_names: list[str], path: str) -> None`
  - `load(path: str) -> list[str]`
  - `validate(features: dict, expected: list[str]) -> None`
- `ModelPredictor`:
  - `load(path: str | Path = "data/models") -> None`
  - `predict(features: dict[str, float]) -> dict[str, float]`

## 7. Database Changes
Изменений в БД нет.

## 8. API Behavior
Публичный формат прогноза сохранен: `{"P1": float, "PX": float, "P2": float}`.

## 9. Business Logic Changes
Контракт инференса стал строгим: модель не принимает несовместимый набор фич, и не пропускает невалидные вероятности в downstream.

## 10. Tests
Добавлены/обновлены проверки:
- сохранение/загрузка/валидация схемы;
- загрузка predictor с calibrated/raw fallback;
- корректность предикта;
- ошибки при schema mismatch;
- ошибки при NaN во входе и NaN/invalid sum/out-of-range в выходных вероятностях;
- сохранение `feature_schema.json` из trainer.

## 11. Manual Verification
Ручная проверка по temporary artifacts в тестах подтверждает:
- `load()` выбирает calibrated модель при наличии;
- fallback на raw работает;
- входной словарь обязан строго совпадать со схемой.

## 12. Risks / Known Issues
- Если схема не сохранена вместе с моделью, `predictor.load()` завершится ошибкой (ожидаемое fail-fast поведение).
- При рассинхронизации `calibrated_model.pkl` и `feature_schema.json` инференс завершится с ошибкой валидации shape/вероятностей.

## 13. Deviations from Spec
В ТЗ есть конфликт пунктов (интеграция в trainer и одновременно «не менять trainer»). Реализована интеграция в trainer, так как это явно требуется в части 2 и необходимо для устойчивого инференса.
