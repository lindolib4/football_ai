# REPORT: predictor-v1-with-calibration

## 1. Summary
Обновлен `ModelPredictor` для приоритетной загрузки `calibrated_model.pkl` (если доступна) с fallback на `model.pkl`, с сохранением строгой валидации входа и вероятностей `P1/PX/P2`.

## 2. Goal
Обеспечить инференс вероятностей матча через калиброванную модель по умолчанию, чтобы использовать более надежные вероятности в downstream-логике.

## 3. Scope
- Изменен `core/model/predictor.py`.
- Обновлены тесты `tests/test_predictor.py`.
- Добавлен отчет этапа.

## 4. Files Changed
- `core/model/predictor.py`
- `tests/test_predictor.py`
- `reports/2026-03-21_16-40_predictor-v1.md`

## 5. Detailed Changes
- `load(path)` теперь принимает путь к директории (по умолчанию `data/models`) или файлу.
- В `load` реализован выбор модели:
  - если есть `calibrated_model.pkl` → загружается она;
  - иначе используется `model.pkl`.
- Добавлены атрибуты состояния загрузки:
  - `raw_model_path`, `calibrated_model_path`, `using_calibrated_model`.
- Схема фич по-прежнему читается от raw-модели (`feature_schema.*`) для стабильного контракта.
- В проверке вероятностей добавлен контроль диапазона `[0, 1]`.

## 6. Public Contracts / Interfaces
`ModelPredictor`:
- `load(path: str | Path = "data/models") -> None`
- `predict(features: dict[str, float]) -> dict[str, float]`

Формат ответа не изменен:
`{"P1": float, "PX": float, "P2": float}`.

## 7. Database Changes
Изменений БД нет.

## 8. API Behavior
Внешний API не менялся; изменение внутреннего выбора артефакта модели.

## 9. Business Logic Changes
Если калиброванный артефакт присутствует рядом с raw-моделью, прогноз делается по калиброванной модели; это повышает согласованность вероятностей для сценариев ставок.

## 10. Tests
Обновлен `tests/test_predictor.py`:
- `test_predictor_uses_calibrated_model_when_available`
- `test_predictor_falls_back_to_raw_model`
- проверки ошибок: missing model, mismatch фич, NaN, invalid sum, out-of-range probability.

## 11. Manual Verification
Проверено вручную на фиктивных моделях:
- при наличии `calibrated_model.pkl` возвращаются вероятности calibrated модели;
- без calibrated файла используется raw модель;
- вероятности выходят в формате `P1/PX/P2`.

## 12. Risks / Known Issues
- Если `calibrated_model.pkl` и `model.pkl` обучены на разных схемах фич, возможны ошибки совместимости при `predict_proba`.
- Проверка фичей основана на схеме рядом с `model.pkl`; это требует консистентности артефактов в одной папке.

## 13. Deviations from Spec
Отклонений по ограничениям нет:
- trainer не изменялся;
- calibrator не изменялся;
- ensemble не добавлялся.
