# REPORT: decision-engine-v1

## 1. Summary
Реализован `DecisionEngine v1` в `core/decision/decision_engine.py`: теперь движок принимает словарь вероятностей (`P1/PX/P2`) и признаки, валидирует вход, считает нормализованную энтропию, gap и confidence, после чего выдает детерминированное решение из множества `1/X/2/1X/X2/12` вместе с метриками.

## 2. Goal
Преобразовывать откалиброванные вероятности матча в логичное и прозрачное betting-решение на основе явных порогов без ML и без randomness.

## 3. Scope

- Переписаны и расширены unit-тесты в `tests/test_decision_engine.py`.
- Добавлен обязательный этапный отчет в `reports/`.

## 5. Detailed Changes
1. Обновлена сигнатура `decide(self, probs: dict, features: dict) -> dict`.
2. Добавлены валидации:
   - обязательные ключи `P1/PX/P2`;
   - отсутствие `NaN`;
   - сумма вероятностей равна 1 (`abs_tol=1e-6`).
3. Реализованы обязательные метрики:
   - `entropy = -sum(p*log(p))`, затем нормализация в `[0,1]` через деление на `log(3)`;
   - `gap = abs(P1 - P2)`;
   - `confidence = max(P1, PX, P2)`.
4. Реализованы явные правила выбора исхода:
   - сильный фаворит (`confidence >= 0.60` и `entropy < 0.85`) → одиночный исход `1/X/2`;
   - средняя неопределенность (`0.40 <= confidence <= 0.60`) → `1X` или `X2`;
   - высокая неопределенность (`entropy >= 0.90` или `gap < 0.15`) → `12`;
   - ничейная зона (`PX >= 0.30` и `PX` близко к максимуму: `<= 0.05`) → включение `X` в виде `1X`/`X2`.
5. Добавлено логирование: `probs`, `entropy`, `gap`, `decision`.

## 6. Public Contracts / Interfaces
Новый публичный контракт:
- Вход: `probs: {"P1": float, "PX": float, "P2": float}`, `features: dict`.
- Выход:
  - `decision`: `"1" | "X" | "2" | "1X" | "X2" | "12"`
  - `confidence`: `float`
  - `entropy`: `float` (нормализованная)
  - `gap`: `float`
